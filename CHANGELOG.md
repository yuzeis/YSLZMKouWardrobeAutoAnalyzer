# Changelog

## ver1.0-beta1 - Gnadenfülle - 2026-07-21

- 启动后自动执行环境检查；环境未就绪时界面直接显示 Npcap、Scapy、网卡或抓包权限的具体原因。
- 启动采集失败提示改为具体环境诊断，不再只要求查看 `runtime/preflight.json`。
- 首个公开 beta 版本。
- 支持 Scapy 与 Npcap 自动选择网卡并被动采集游戏流量。
- 实时显示卡池、抽卡记录、完整衣柜和背景页面的阅览状态。
- 从完整衣柜生成覆盖全部普通池与背景池的微信小程序导入数据。
- 输出原始 JSON、J1、C1 Base64、C1 Base4096 及可选内容二维码。
- 启动即请求管理员权限；抓包与报告实时落盘，导出完成后自动清理 PCAP。
- 增加三段启动声明、环境检查、冻结烟测与 Windows 单文件发行配置。
- 修复新版游戏静态目录格式变化导致微信导入生成失败；完整衣柜可使用冻结小程序目录保守生成。
- 原始微信导入 JSON 改为优先显示并落盘，压缩格式或二维码失败时仍可复制、保存和导入。
- 适配 2026-07-21 当前游戏包的前置 BCFG 表布局，静态服装目录更新为 Fashion v9/9530、FashionSuite v2/943。
